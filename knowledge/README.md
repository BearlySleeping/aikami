# Aikami Knowledge

AI-first knowledge hub for the Aikami project.

**This is the knowledge layer of the monorepo.** It holds specifications, decisions, design references, and guides. AI tools (pi, Claude) read and write here. All knowledge lives alongside the code in a single monorepo.

## Structure

| Folder | What's inside |
|--------|--------------|
| `contracts/` | Feature specifications with data models and acceptance criteria |
| `decisions/` | Architecture Decision Records (ADRs) |
| `design/` | Design specs, tokens, and visual references |
| `architecture/` | System architecture and technical docs |
| `guides/` | How-to guides and workflows |
| `intro/` | Vision, setup, and developer onboarding |
| `tickets/` | Bug reports and feature requests |
| `scripts/` | Knowledge maintenance scripts (generate_llms_txt.ts) |

## For AI Tools

1. Read `llms.txt` first — it's the map
2. Read `CONTEXT.md` — 2-page briefing
3. Find 2-5 relevant files, read them, then write

## For Humans

Browse on GitHub — all markdown renders natively. Start with `CONTEXT.md` for a high-level overview, then dive into specific folders based on what you need.
