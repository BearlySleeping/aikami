# Contract: C-031 SvelteKit Adapter Static Conversion & Firebase Hosting Setup

## Design References

- Monorepo Boundaries: `.moon/workspace.yml`
- PWA Configuration: `apps/frontend/client/svelte.config.js` and `apps/frontend/client/vite.config.ts`
- Firebase Configurations: `firebase.json` and `apps/backend/firebase/firestack.config.ts`

## Detailed Changes

### 1. SvelteKit Adapter Migration

- **Dependency Swap**: Remove `svelte-adapter-bun` from `apps/frontend/client/package.json`. Add `@sveltejs/adapter-static`.
- **Config Overhaul**: Modify `svelte.config.js` to clear out the Bun server configuration and initialize `adapter-static`.
- **SPA Fallback Configuration**: Configure the static adapter to produce a pure Single Page Application (SPA) output by setting the fallback parameter to `index.html`.
- **Global Layout Forcing**: Create or update `apps/frontend/client/src/routes/+layout.ts` to explicitly export `const ssr = false;` and `const prerender = false;`, guaranteeing that the layout is treated as a client-side wrapper.

### 2. Firebase Hosting Alignment

- **`firebase.json` Structuring**: Point the hosting site configuration directly to the static build asset output directory generated by SvelteKit (`apps/frontend/client/build`).
- **SPA Rewrites**: Add a catch-all rewrite rule mapping `/**` to `/index.html` inside `firebase.json` so that deep client-side paths resolve properly without returning 404 errors.
- **Emulator Enforcement**: Include the `hosting` target within the local emulator configuration block under `firebase.json`.

## Acceptance Criteria

- **Given** the PWA has been compiled with adapter-static.
- **When** executing `moon run client:build`.
- **Then** the output contains no server dependencies and outputs cleanly to `apps/frontend/client/build/index.html`.
- **Given** the local emulator environment is active.
- **When** triggering the emulator start sequence via `bun run dev:all`.
- **Then** the Firebase Emulator Suite spins up a local hosting server on port `5000` (or your configured port) serving the built assets.

## Watch Points

- **Client-Side Svelte Hooks**: `src/hooks.server.ts` will no longer intercept incoming runtime traffic during client operation. Ensure critical auth-check guards are mapped into layout initializers (`+layout.ts`) or ViewModels.
- **Cache-Control Overrides**: In development, make sure the Firebase emulator does not aggressively cache the `index.html` file so that your hot reloading or manual refreshes render updated code instantly.
