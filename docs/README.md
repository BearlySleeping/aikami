# Aikami Documentation

Human-facing documentation for the Aikami project. AI tooling files live in `.context/`.

## Structure

| Folder | What's inside |
|--------|--------------|
| `architecture/` | System architecture and technical docs |
| `contracts/` | Feature specifications with data models and acceptance criteria |
| `decisions/` | Architecture Decision Records (ADRs) |
| `design/` | Design specs, tokens, and visual references |
| `guides/` | How-to guides and workflows |
| `intro/` | Vision, setup, and developer onboarding |
| `scripts/` | Knowledge maintenance scripts documentation |
| `tickets/` | Bug reports and feature requests |

## For AI Tools

1. Read `.context/llms.txt` first — it's the map
2. Read `.context/CONTEXT.md` — 2-page briefing
3. Find 2-5 relevant files, read them, then write

## For Humans

Browse on GitHub — all markdown renders natively. Start with `.context/CONTEXT.md` for a high-level overview, then dive into specific folders based on what you need.
